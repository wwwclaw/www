Whiteboard Capture Projects

I still need a cool name for these two projects. The overall goal is to make it easy to capture a whiteboard drawings. I feel like drawing something digitally hampers the flow of ideas, you either use a tool that slows you down, or you easily get distracted and start polishing what's on the screen. For most people, using pen and paper or working with a whiteboard is the easiest way to brainstorm and be creative.

Cleanup

The first tool cleans up a whiteboard image. The original version followed a research paper from Microsoft, but I have since rewritten every step using different techniques. The new tool is equally fast and can better deal with glare. This tool can output a bitmap, or vectorize the outline of each mark on the whiteboard.



Processing

The second tool extracts the original marker-strokes from the whiteboard image. This is best demonstrated with an example:
It is straightforward to get the outline of a mark on the whiteboard - i.e. finding the outline of the white region. Take the letter "I" - the outline is a vertical rectangle, even though it takes a vertical line to draw the "I" with a marker.

Now hover or touch the Jurassic Park logo. Calculating the red lines based on the white outlines is hard - well maybe not for the "I", but it's really tricky for the "K". That's what this project does.

After getting the original marker coordinates, the points have a lot in common with the data from a touch gesture. Tools written to work with touch devices can now work with pictures! The proof of concept is works, though it is very rough, and simple whiteboard images can connect to an old project of mine, GrafPad.

GrafPad

GrafPad is a drawing tool designed for touch devices. The idea is to act like a digital pen and paper. The input is free form - either touch or mouse inputs, and the tool recognizes objects and automatically cleans them up. So lines that are near horizontal or vertical are made perfectly horizontal or vertical, and polygons are recognized and normalized wherever possible (opposite sides set to equal size, angles converted to 90 degrees, etc).

I wrote the backend, including designing and building all the cleanup algorithms. You can see it in action below.

RapydScript I

This project is actually the brainchild of Alex Tsepkov, but I was involved in building and testing the original code, so I thought it worth mentioning.

I call RapydScript a Python to JavaScript transpiler, as opposed to a compiler. It is similar to CoffeeScript, but more Pythonic. RapydScript has a Pythonic syntax but is JavaScript under the hood. It adds a few things like inheritance to make building webapps easier.

I helped figure out how to design a lot of the syntax where things weren't so clear, and created the first implementation of several key features like tuple packing/unpacking and exception handling. I also built one of the largest initial proof of concepts, Cyborg Challenge, which is a RapydScript clone of Chips Challenge that you can play online.
